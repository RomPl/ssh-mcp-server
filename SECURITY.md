# Политика безопасности

## Сообщение об уязвимости

Не заводите публичный issue с подробностями уязвимости. Используйте приватные каналы (Security Advisories / личный контакт владельца репозитория).

## Практики безопасности (пользователю)

1) Секреты
- Никогда не коммитить `.env`.
- Права на файл: `chmod 600 .env`.
- Секреты должны быть длинными случайными строками.

2) OAuth
- `ALLOWED_EMAIL` должен быть вашим email.
- `PUBLIC_BASE_URL` должен соответствовать внешнему HTTPS URL сервера.
- Для Custom GPT Actions static client (`chatgpt`) без PKCE: включайте только через env и только на один `client_id`.

3) SSH
- Предпочтительна key-based аутентификация.
- Ограничьте команды через `ALLOWED_COMMANDS`, если требуется.

## Практики безопасности (разработчику)

- Не логировать секреты/заголовки авторизации.
- На всех защищённых endpoint’ах проверять JWT (Bearer).
- Не ослаблять PKCE глобально: исключение допустимо только для одного static клиента.
